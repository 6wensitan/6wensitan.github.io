当然也可以在SqlSessionFactoryBuilder.build（）方法中传入参数。例如：

如果一个属性在不止一个地方进行配置,那么,Mybatis将按照下面的顺序来加载:

  • 首先将读取properties元素体内指定的属性.
  • 然后根据文件中的resource属性读取类路径下的属性文件,或者根据url属性指定的路径读取文件,并覆盖之前读取过的同名属性.
  • 最后读取作为方法参数传递的属性,并覆盖之前读取过的同名属性.

因此,**通过方法参数传递的属性具有最高优先级,**resource/url属性中指定的配置文件次之,最低优先级原则是properties元素指定的元素.

mybatis不会通过监测数据库元信息来决定使用的类型, 所以参数和结果映射要指明varchar类型. 这样可以准确绑定.

原因: mybatis是只有执行语句的时候才知道数据类型

那我们该怎么让mybatis知道呢? 答: 类处理器的泛型.

两种方法改变上面的设置:

  • 在类型处理器的配置元素上增加一个JavaType属性(例如: JavaType=”String”)

  • 在类处理器上增加一个注解(@MappedJdbcTypes)指定关联的数据类型.

  • 注意: 如果在方法一上也指定了, 注解上的会失效.

dataSource数据源

数据源使用标准的jdbc数据源接口来配置jdbc连接对象的资源.(如果要启用延迟加载特殊性,必须配置数据源)

三种内置数据源:

UNPOOLED

实现该数据源会每次请求时打开和关闭链接,有点慢,仅需要配置以下五个属性:

  • driver -数据库驱动: Java类全限定名字
  • url -数据库jdbc的url地址
  • username -数据库名字
  • password -数据库密码
  • defaultTransactionLevel -默认的连接事务隔离级别
  • defaultNetworkTimeout -等待数据库操作的默认网络超时

可选项: 可以传递属性给数据库驱动,在属性名前添加driver.前缀即可

driver.encoding=UTF8

如何将该属性传递呢?

答: 通过DriverManager.getConnection(url,driverProperties)方法传递值为UTF8的encoding属性给数据库驱动

POOLED

利用池化的概念链接jdbc,避免创建新的连接实例的初始化和时间.

可以并发web应用快速响应请求.

属性:其实和上面的一样,但是也有其他 属性:

比如:

  • 限制连接个数: poolMaximumActiveConnections(默认值:10)
  • 检查可能存在的空连接数: poolMaximumIdleConnections
JND

实现在容器中去使用的

只用配置两个属性:

  • initial_context: 用来在initialContext寻找上下文,(可选顺序ing)
  • data_source : 引用数据实例的地址.
  • 如果像引用其他属性的话,方法和数据源一一样,只是改为env.前缀

针对于单个参数传递的封装

  • pojo类型: 直接使用,属性名和参数占位符名称一致
  • map集合: 直接使用, 键名和参数占位符名称一致
  • collection: 封装为map集合, 可以使用@Param注解, 替换map集合默认的键名(arg0)
    • map.put(“arg0”,collection集合);
    • map.put(“collection”,collection集合);
  • List集合: 封装为map集合, 可以使用@Param注解, 替换map集合默认的键名(arg0)
    • map.put(“arg0”,list集合);
    • map.put(“collection”,list集合);
    • map.put(“list”,list集合);
  • Array数组: 封装为map集合, 可以使用@Param注解, 替换map集合默认的键名(arg0)
    • map.put(“arg0”,数组);
    • map.put(“array”,数组);
  • 其他类型: 直接使用(如: int id)
