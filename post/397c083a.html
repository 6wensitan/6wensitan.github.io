Java8新特性

发表于2023-08-31|更新于2023-09-03|jiepu
|字数总计:2.3k|阅读时长:11分钟

前言

  • 笔记: 新特性/mysql/jdbc/svn/maven
  • 整理人: vanse(lc)
  • 时间: 2023/8/29

新特性

接口方法

  • 默认方法 default 实现类调用
  • 静态方法 static 接口调用

==函数式编程==

通过lambda简化匿名内部类(实现的接口必须是函数式接口(只有一个抽象方法的接口))

  • 参数 (a,b) -> {}
  • 返回值 {return }
  • 代码逻辑 {return a + b}
    • 如果代码逻辑只有一行 (a,b) -> a+b

函数式接口

内置了以下函数式接口,无需自定义函数式接口

  • 断言型接口 Predicate<T> 一般用于过滤数据
    • boolean test(T t)
  • 消费型接口 Consumer<T> BiConsumer<T,E> 一般用于遍历数据
    • void accept(T t)
    • void accept(T t,E e)
  • 生产型接口 Supplier<T>
    • T get();
  • 转换型接口 Funcation<T,R>
    • R apply(T t) 将第一个类型转换为第二个类型

==四个方法的参数和返回值特征决定使用哪个函数式接口==

步骤

  1. 抽取一个方法
  2. 方法中先用函数式接口的抽象方法 表明某种含义
  3. 提供函数式接口的实现类,重写抽象方法
  4. 调用方法,传递接口实现类

方法引用

可以不用掌握,简化lambda

​ 别人写好的方法的 参数和返回值和 函数式接口中抽象方法 参数返回值近似时,可以引用别人的方法

  • 静态方法引用

    • 静态方法
    • 参数和返回值满足
    • 类::静态方法引用 Integer::parseInt

  • 实例方法引用

    • 对象引用实例方法引用

      • 普通方法
      • 参数和方法满足

    • 类引用实例方法

      • 抽象方法: 必须第一个参数类型为隐形的对象,实际实现类的方法参数比抽象方法的参数少一个
      • 该对象的方法的参数和返回值与抽象方法一致即可

  • 构造方法引用

    • 无参构造器 Teacher::new
    • 有参构造器 Teacher::new
    • 数组构造器 int[]::new

Optional

使用场景: 包装对象,防止空指针

==Stream==

使用尝尽: 流式编程 一次对数据做多个操作

数据->流

流->数据

流方法

原始数据 ->加工1 ->加工2 -> 最终数据 -> 不能再加工

最终方法 -> 最终数据 不能再加工

中间方法 -> 流 -> 流 -> 最终fang’fa
