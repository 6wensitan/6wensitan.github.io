二、IO流

1、基础问答

根据数据的流向，可以将流分为哪些类型？简述它们的特点。

  • 根据数据的流向，流可以分为以下两种类型：

    1. 输入流（Input Stream）：

      输入流用于从外部数据源（如文件、网络、键盘等）读取数据到程序中。它提供了从数据源读取数据的方法，通常用于读取数据。

      特点：

      • 输入流是单向的，只能读取数据，不能写入数据。
      • 输入流通常以字节或字符为单位读取数据。
      • 输入流通常用于从文件、网络、标准输入等读取数据。

    2. 输出流（Output Stream）：

      输出流用于将程序中的数据写入到外部目标（如文件、网络、屏幕等）。它提供了将数据写入目标的方法，通常用于写入数据。

      特点：

      • 输出流是单向的，只能写入数据，不能读取数据。
      • 输出流通常以字节或字符为单位写入数据。
      • 输出流通常用于将数据写入文件、网络、标准输出等。

2、基础问答

根据每次处理数据的单位，可以将流分为哪些类型？简述它们的特点。

  • 根据每次处理数据的单位，流可以分为以下两种类型：

    1. 字节流（Byte Stream）：

      字节流以字节为单位进行数据传输，适用于处理二进制数据，如图片、音频、视频等。字节流通常用于读取和写入文件、网络传输等二进制数据操作。

      特点：

      • 字节流适用于处理任意类型的数据，包括文本和二进制数据。
      • 适用于处理二进制文件，如图片、音频等。
      • 由于以字节为单位处理，适合处理任意格式的数据。

    2. 字符流（Character Stream）：

      字符流以字符为单位进行数据传输，适用于处理文本数据，如文本文件。字符流会自动处理字符编码转换，确保文本的正确读写。

      特点：

      • 字符流适用于处理文本数据，可以自动处理字符编码转换，避免乱码问题。
      • 适用于处理纯文本文件，如文本文档、配置文件等。
      • 以字符为单位处理，适合处理文本格式的数据。

    通常情况下，如果处理的是文本数据，使用字符流能够更好地处理字符编码和字符转换问题，而如果处理的是二进制数据，使用字节流更合适。

3、基础问答

JDK 为每种类型的流提供了抽象类以定义标准和规范，请说出他们分别是哪些类？

  • JDK 为每种类型的流提供了抽象类以定义标准和规范。以下是一些常见的流类型及其对应的抽象类：

    1. 字节流抽象类：

      • InputStream：字节输入流的抽象类，定义了读取字节的标准接口。
      • OutputStream：字节输出流的抽象类，定义了写入字节的标准接口。

    2. 字符流抽象类：

      • Reader：字符输入流的抽象类，定义了读取字符的标准接口，同时提供了字符编码支持。
      • Writer：字符输出流的抽象类，定义了写入字符的标准接口，同时提供了字符编码支持。

    3. 缓冲流抽象类：

      • BufferedReader：字符缓冲输入流的抽象类，提供了缓冲读取字符的能力，增加了读取效率。
      • BufferedWriter：字符缓冲输出流的抽象类，提供了缓冲写入字符的能力，增加了写入效率。

    4. 数据流抽象类：

      • DataInputStream：数据输入流的抽象类，用于读取基本数据类型。
      • DataOutputStream：数据输出流的抽象类，用于写入基本数据类型。

    5. 对象流抽象类：

      • ObjectInputStream：对象输入流的抽象类，用于读取对象的序列化表示。
      • ObjectOutputStream：对象输出流的抽象类，用于写入对象的序列化表示。

4、基础问答

InputStream类中定义了三个⽤于读取字节数据的⽅法，请回答以下问题：

1、简述每个⽅法的作⽤以及参数的含义

2、三个⽅法均为int类型返回值，这三种返回值代表的含义有何异同点？

​ int read();

​ int read(byte b[]);

​ int read(byte b[], int off, int len);

  • 当涉及到字节数据的读取，InputStream类中提供了三个方法，下面对每个方法的作用和参数含义进行简述：

    1. **int read()**：

      • 此方法从输入流中读取一个字节的数据。它将下一个字节的数据作为整数值返回（范围为0-255），如果已经到达流的末尾，则返回-1。
      • 无需参数。

    2. **int read(byte b[])**：

      • 此方法将一系列字节数据读入到给定的字节数组b中。它会读取与数组长度相等的字节数，或者读取到流的末尾为止。它会返回读入到数组中的字节数，如果已经到达流的末尾，则返回-1。
      • 参数：
        • b：用于存储读取数据的字节数组。

    3. **int read(byte b[], int off, int len)**：

      • 此方法将最多len个字节的数据读入到给定的字节数组b中，从数组中的偏移量off处开始。它会返回读入到数组中的字节数，如果已经到达流的末尾，则返回-1。
      • 参数：
        • b：用于存储读取数据的字节数组。
        • off：字节数组b中的起始偏移量。
        • len：最大要读取的字节数。

    这三个方法的返回值类型都是int，其含义有所不同：

    • 对于这三个方法，如果它们成功读取了一个或多个字节，它们将返回实际读取的字节数。
    • 如果已经到达流的末尾，它们都会返回-1，表示没有更多的数据可供读取。

    总结：

    • read()：读取单个字节数据。
    • read(byte b[])：将字节读入提供的数组。
    • read(byte b[], int off, int len)：将字节读入提供的数组，可以指定偏移量和长度。

5、基础问答

请回答在什么情况下适合⽤字节流读写⽂件？什么情况下适合⽤字符流读写⽂件？

  • 适合使用字节流读写文件的情况：

    1. 处理二进制文件：字节流适用于读写任何类型的数据，包括二进制文件，如图片、音频、视频等。

    2. 网络传输：在网络传输中，数据往往是以字节的形式传输的，因此字节流适用于处理网络数据。

    3. 未经处理的文件：当需要对文件中的内容不加处理地读写时，字节流是更合适的选择。

    适合使用字符流读写文件的情况：

    1. 处理文本文件：字符流适用于处理文本文件，可以自动处理字符编码转换，避免乱码问题。

    2. 文本处理需求：当需要对文本文件进行按字符的读写、处理和转换时，字符流是更合适的选择，可以更方便地处理字符级操作。

    3. 自动处理字符编码：字符流会自动处理字符编码，因此在需要确保正确的字符编码转换时，使用字符流可以避免编码问题。

    综上所述，如果处理的是二进制数据、网络数据或者不经过特殊处理的数据，字节流是更合适的选择。而如果处理的是文本数据，需要字符编码转换或者字符级处理，字符流则更加适合。实际应用中，根据需求选择适当的流类型能够更好地满足处理数据的要求。

6、基础问答

内存缓冲流的优势有哪些？

  • 内存缓冲流（BufferedInputStream 和 BufferedOutputStream）在处理输入输出时引入了缓冲区的概念，从而带来了一些优势：

    1. 提高读写效率：内存缓冲流通过在内存中创建缓冲区，允许以块的方式进行数据读写，从而减少了频繁的单字节或单字符的读写操作，提高了读写效率。

    2. 减少系统调用：内存缓冲流允许程序以块的方式读写数据，这减少了与底层操作系统的交互次数，减少了系统调用的开销，从而提高了整体性能。

    3. 减少磁盘访问：对于磁盘IO，内存缓冲流可以将多个小的读写操作合并为一个大的读写操作，减少了磁盘访问次数，降低了磁盘IO的开销。

    4. 适用于大文件处理：内存缓冲流在处理大文件时特别有效，它可以将大文件分成块进行读写，避免一次性将整个大文件加载到内存中造成内存溢出。

    5. 适用于网络传输：在网络传输时，内存缓冲流可以在内存中暂存数据，然后一次性发送或接收一块数据，降低了频繁的网络IO操作。

    总之，内存缓冲流通过引入缓冲区的概念，可以有效地提高数据读写的效率，减少系统调用和磁盘访问，适用于大文件处理和网络传输等场景。
