线程休眠考核

需求分析：

处于运行状态的线程在某些情况下，如执行了sleep（睡眠）方法，或等待I/O设备等资源，将让出CPU并 暂时停止自己的运行，线程将进入什么状态。如何重新进入到 RUNNABLE 状态？

  • 问答一:

    • 当一个处于运行状态的线程执行了 sleep 方法、等待 I/O 设备等资源时，它会进入到 阻塞状态（Blocked）或 等待状态（Waiting）。

  • 问答二:

    • 阻塞状态（Blocked）:当其他线程==释放锁==并唤醒了该线程时，该线程会进入到就绪状态（RUNNABLE）。
    • 等待状态（Waiting）:当其他线程通过 Object.notify() 或 Object.notifyAll() 唤醒了等待的线程，或者 join 的线程执行完毕，或者等待的时间到达时限，等待的线程会进入到就绪状态（RUNNABLE）

    需要注意的是，就绪状态并不保证线程会立即执行，而是由==线程调度器==来决定线程何时获得 ==CPU 时间片==并开始执行。

程序分析

需求分析：

在线程A中调用线程B的join()方法会产生什么效果？

  • 等待线程 B 执行完成： 当线程 A 调用线程 B 的 join() 方法时，线程 A 将会进入等待状态，直到线程 B 执行完成（即线程 B 正常终止或抛出异常）。在这个过程中，==线程 A 会被阻塞，不会继续执行，直到线程 B 完成为止==。
  • 线程合并： 调用 join() 方法实际上是将线程 A 合并到线程 B 中，让线程 B 的执行变成了线程 A 所在的代码执行。这样可以等待线程 B 执行完毕后，线程 A 才会继续执行。

==实现多个线程协同工作的场景==通过 join() 方法，主线程可以等待子线程完成，确保在需要子线程结果的时候，已经得到了正确的结果。

线程安全

需求分析：

什么是线程安全？

  • 线程安全（Thread Safety）是指在多线程环境下，==共享的资源==（如数据、对象、变量等）能够被多个线程==同时访问==而不引发不正确的结果或不一致的状态。一个线程安全的程序能够保证在==并发情况下==，不会发生数据竞争、数据损坏、死锁等问题，从而保障程序的正确性和可靠性。
  • 常见的线程安全策略包括：==(避免共享资源竞争)==
    • 互斥锁(Mutex): 通过锁来保护共享资源，确保在同一时间只有一个线程能够访问修改资源。
    • 不可变对象: 如果共享资源不可修改，那么多个线程访问也不会产生竞争条件。
    • 线程局部变量(Thread-Local Variables): 每个线程拥有独立的变量实例，避免了共享资源的竞争。
    • 并发容器:提供了线程安全的数据结构，如 ConcurrentHashMap、CopyOnWriteArrayList ..

方法区分

需求分析：

描述wait()方法、sleep()方法和join()3个方法的异同点

  • 相同点:

    • 阻塞线程: 所有这三个方法都可以使线程==进入阻塞状态==，即暂停线程的执行。
    • 异常抛出: 它们都可能抛出 InterruptedException 异常，因此在使用时需要进行异常处理。

  • 不同点:

    • 用途

      • wait() 方法：用于==线程之间的协调和通信==，它通常在等待某个条件满足时释放锁，并让线程进入等待状态，

                                 等待其他线程通过 `notify()` 或 `notifyAll()` 唤醒它。

      • sleep() 方法：用于==暂停线程的执行一段指定的时间==，不释放任何锁。它可以用于暂时释放 CPU 资源，让其他线程有机会执行。

      • join() 方法：用于==等待其他线程完成==。当一个线程调用另一个线程的 join() 方法时，调用线程会阻塞，

        ​ 直到被调用线程执行完成。

    • 调用方式和对象

      • wait() 方法：调用对象是一个锁对象，即必须在同步代码块或同步方法中使用。
      • sleep() 方法：可以在任何地方调用，不需要持有锁。
      • join() 方法：调用对象是其他线程，用于等待调用它的线程执行完成。

    • 是否释放锁

      • wait() 方法：在等待时会==释放对象的锁==，允许其他线程进入同步块或方法。
      • sleep() 方法：在休眠期间不会释放锁，==其他线程无法进入同步块或方法==。
      • join() 方法：调用它的线程会等待被调用线程执行完成，但是==不会释放锁==。

    • 时间参数

      • wait() 方法：可以指定==等待的超时时间==。
      • sleep() 方法：必须指定休眠的时间。
      • join() 方法：可以指定等待的超时时间。

  • 总结:wait() 主要用于线程间的通信，sleep() 用于线程的暂时休眠，join() 用于等待其他线程完成。

线程同步

需求分析：

简述synchronized修饰静态方法和非静态方法的区别

  • 非静态方法
    • 当一个非静态方法使用 synchronized 修饰时，称为同步实例方法。==实例对象级别的锁==
    • 其他线程在访问同一类的同步静态方法时，需要等待锁的释放。（同）
    • 同步实例方法之间也可以实现同步，因为它们共享实例对象的锁。==实例方法之间互相有影响==
  • 修饰静态方法
    • 当一个静态方法使用 synchronized 修饰时，称为同步静态方法。静态方法是类级别，==类级别的锁==
    • 其他线程在访问同一类的同步静态方法时，需要等待锁的释放。（同）
    • 同步静态方法和同步实例方法之间不会相互影响，因为它们使用的是不同的锁。==静态方法和实例方法之间互不影响：锁不同==

银行转账

需求分析：

假设现在有两个用户账户A和B，每个账户都有一定的余额，现在需要实现一个线程安全的转账功能，即 一个用户可以将一定的金额转入到另一个用户账户中。

要求如下：

  1. 用户可以同时进行多次转账操作，每次转账的金额必须大于0，同时转出账户余额必须大于等于转 账金额。

  2. 转账过程中，需要对转出账户和转入账户加锁，保证同时只能有一个线程对某个账户进行转账操 作。

  3. 在进行转账操作时，需要按照账户名称的字典序进行排序，保证转账过程中的资源占用顺序一 致，避免死锁的出现。

  4. 转账操作完成后，需要释放相应的锁，并输出转账的金额和最新的账户余额。

  5. 要求设计一个银行账户转账类，包含以下方法：

    transfer(Account from, Account to, double amount) ：从账户from中转出一定的金额， 将其转入到账户to中。

    其中，Account是一个用户账户类，包含以下属性：

    ​ String name ：账户名称。 double balance ：账户余额。

    编写测试程序，模拟多个用户同时进行转账操作，验证银行账户转账类的线程安全性

  • //Account类
public class Account {
    private String name; // 账户名称
    private double balance; // 账户余额

    public Account(String name, double balance) {
        this.name = name;
        this.balance = balance;
    }

    // 存款
    public void deposit(double amount) {
        balance += amount;
    }

    // 取款
    public void withdraw(double amount) {
        balance -= amount;
    }

    public String getName() {
        return name;
    }

    public double getBalance() {
        return balance;
    }

    // 转账
    public void transfer(Account to, double amount) {
        // 按照账户名称的字典序进行排序，保证转账过程中的资源占用顺序一致，避免死锁的出现
        Account firstAccount = (name.compareTo(to.getName()) < 0) ? this : to;
        Account secondAccount = (name.compareTo(to.getName()) < 0) ? to : this;

        synchronized (firstAccount) {
            synchronized (secondAccount) {
                // 转出账户余额必须大于等于转账金额
                if (balance >= amount) {
                    withdraw(amount);
                    to.deposit(amount);
                    System.out.println(Thread.currentThread().getName() + " transferred " + amount + " from " + name + " to " + to.getName() + ". " + name + " Balance: " + balance + ", " + to.getName() + " Balance: " + to.getBalance());
                } else {
                    System.out.println(Thread.currentThread().getName() + " failed to transfer " + amount + " from " + name +" to " + to.getName() + ". Insufficient balance.");
                }
            }
        }
    }
}

什么是线程通信

  • 含义:线程通信是多线程编程中的一个概念，指的是在多个线程之间传递信息、数据或信号，

    ​ 以协调它们的执行顺序和共享资源的访问，以实现某种协作目标。

  • 常见的线程通信机制包括：

    1. wait() 和 notify() / notifyAll() 方法：这是基于监视器锁（synchronized）的机制，通过调用 wait() 方法使线程等待，然后通过 notify() 或 notifyAll() 方法来唤醒等待的线程。

    2. Condition：在 Java 的 java.util.concurrent 包中提供了 Condition 接口，它可以创建多个等待队列，每个队列可以用于不同的条件，从而更细粒度地进行线程通信。

    3. BlockingQueue：阻塞队列是一种线程安全的队列实现，在队列为空时会阻塞等待，或者在队列满时阻塞插入操作，可以用于线程之间的数据传递和协调。

    4. Semaphore：信号量是一种计数信号量，可以用来控制同时访问某个资源的线程数，常用于控制资源的并发访问。

    5. CountDownLatch 和 CyclicBarrier：这些类也是用于线程协作和通信的工具，CountDownLatch 可以实现等待一组线程都完成后再继续执行，而 CyclicBarrier 可以在多个线程达到同步点时进行等待。

    线程通信的目的是确保多个线程之间的正确协作，以避免数据竞争、死锁、饥饿等问题，从而实现安全和高效的多线程编程。

public class Test_09 {
    public static void main(String[] args) {
        Bun_Shop bs = new Bun_Shop();
        
        Thread make1 = new Thread("师傅") {
            @Override
            public void run() {
                while(true) {
                    bs.make();
                }
            }
        };
        Thread buy1 = new Thread("小明") {
            public void run() {
                while(true) {
                    bs.buy("蔬菜馅");
                }
            }
        };
        Thread buy2 = new Thread("小红") {
            public void run() {
                while(true) {
                    bs.buy("肉馅");
                }
            }
        };
        Thread buy3 = new Thread("小白") {
            public void run() {
                while(true) {
                    bs.buy("蔬菜馅");
                }
            }
        };
        
        make1.start();
        buy1.start();
        buy2.start();
        buy3.start();
    }
    
    static class Bun_Shop{
        private static int bunCount=0;
        private static int meatBunCount=0;
        private static int verBunCount=0;
        
        public void make() {
            synchronized (Bun_Shop.class) {
                System.out.println(Thread.currentThread().getName()+"制作包子中");
                try {
                    Thread.sleep(10);
                    
                } catch (InterruptedException e) {
                    // TODO Auto-generated catch block
                    e.printStackTrace();
                }
                if(bunCount%2==0) {
                    verBunCount++;
                }else {
                    meatBunCount++;
                }
                bunCount++;
                System.out.println("包子做好啦......现在总共有:" + bunCount+",  蔬菜包有:"+verBunCount+",  肉包子有:"+meatBunCount);
                Bun_Shop.class.notifyAll();
            }
        }
        
        public void buy(String taste) {
            synchronized (Bun_Shop.class) {
                while (bunCount == 0) {
                    System.out.println(taste + "的包子没有啦,等等....");
                    try {
                        Bun_Shop.class.wait();
//						notifyAll();
                    } catch (InterruptedException e) {
                        // TODO Auto-generated catch block
                        e.printStackTrace();
                    }
                }
                if(taste.equals("蔬菜馅")) {
                    if(verBunCount>0) {
                        verBunCount--;
                    }
                }else {
                    if(meatBunCount>0) {
                        meatBunCount--;
                    }
                }
                bunCount--;
                System.out.println( Thread.currentThread().getName()+ "买到  "+taste+"  的包子啦啦......" + "当前包子还有: " + bunCount+",  蔬菜包有:"+verBunCount+",  肉包子有:"+meatBunCount);
                
            }
        }
    }
}
