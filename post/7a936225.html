简述throw 和 throws 的区别 【提示】 可从以下⼏个⽅⾯思考： 声明位置\ 抛出异常的个数 \关键字作⽤

throw 和 throws 是 Java 中用于异常处理的两个关键字，它们具有不同的作用和用法：

  1. 声明位置：

    • throw 关键字用于在方法内部抛出一个异常对象。它通常用于在方法内部发现了某种错误或异常情况，然后显式地抛出一个异常对象来中断方法的正常流程。
    • throws 关键字用于方法签名中，表示该方法可能会抛出的异常类型。在方法声明时，可以使用 throws 关键字列出方法可能抛出的受检查异常，以便调用者知道可能需要处理的异常。

  2. 抛出异常的个数：

    • throw 关键字在方法内部通常用于抛出一个单独的异常对象。每次使用 throw，只能抛出一个异常。
    • throws 关键字在方法签名中可以列出多个可能抛出的异常类型，这些异常类型将作为方法的受检查异常。

  3. 关键字作用：

    • throw 关键字用于在方法内部抛出异常，将异常传递给方法的调用者。它常用于手动创建并抛出异常对象，以便在方法内部处理异常情况。
    • throws 关键字用于方法声明，表示该方法可能抛出的异常类型。它是方法签名的一部分，通知调用者需要考虑和处理可能的异常情况。

总之，throw 用于==抛出异常对象==，而 throws 用于在==方法签名中==声明可能抛出的异常类型。它们在异常处理机制中扮演不同的角色。

finally问答

finally代码块是否⼀定会执⾏

  • 在大多数情况下，finally 代码块是会执行的。finally 块中的代码会在以下情况下被执行：

    1. 当相应的 try 块完成执行时，无论是否发生了异常。
    2. 当异常被抛出且在当前方法内没有被捕获时。

    无论是 try 块内部的代码正常执行结束还是发生了异常，finally 块中的代码都会被执行。

    这使得 finally 块适合用来进行一些资源的释放、清理操作，以确保在不同的代码路径下都能够进行必要的收尾工作。

    然而，有一些情况下 finally 块可能不会执行，例如：

    1. 在执行 try 块的过程中，如果发生了死锁、线程被中断或程序被非正常终止等情况，

      可能导致 finally 块无法执行。

    2. 在某些情况下，System.exit() 调用会导致程序直接退出，此时 finally 块可能不会执行。

    总之，虽然大部分情况下 finally 块都会执行，但也应该考虑到在特殊情况下可能会导致 finally 块不被执行的情况。

值传递和参数传递:

值传递（Pass by Value）：
在值传递中，==方法或函数接收的是实际参数的副本，而不是原始参数本身==。这意味着在方法内对参数的修改不会影响到调用者传递的原始参数。在 Java 中，无论是基本数据类型还是引用类型，都是以值传递的方式传递给方法。

对于基本数据类型（如 int、float、char 等），传递的是实际的值的拷贝，因此对参数的修改不会影响原始值。对于引用类型（如对象、数组、集合等），传递的是引用的值的拷贝，即引用的副本，因此在方法内可以修改引用所指向的对象的状态，但不能改变引用本身，也不能改变调用者传递的引用所指向的对象。==String是final修饰,不可更改,,地址改不了,内容也改不了.==

：Java 中传递参数的机制是值传递，对于基本数据类型和引用类型，传递的都是值的拷贝，但在引用类型的情况下，拷贝的是引用的值（即引用的副本）。这一机制确保了传递的参数不会被方法内部的修改所影响。
